## Multi-tenancy

- Estrategia: **shared database, shared schema con `tenant_id`** en todas las tablas.
- Refuerzo de aislamiento: **Row-Level Security (RLS) de PostgreSQL**.
- En el backend, todo request autenticado debe inyectar `tenant_id` en el contexto vأ­a un guard/middleware. Las consultas Prisma DEBEN filtrar siempre por `tenant_id` (usar una extensiأ³n Prisma o un repositorio base).
- NUNCA exponer endpoints que devuelvan datos sin filtrar por tenant.
